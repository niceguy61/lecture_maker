# AWS Console 기본 탐색 가이드

## 📋 개요

이 가이드는 AWS Management Console의 기본 구조와 탐색 방법을 안내합니다. Console의 주요 메뉴, 검색 기능, 리전 선택, 즐겨찾기 설정 등을 학습하여 효율적으로 AWS 서비스를 사용할 수 있습니다.

## 🎯 학습 목표

- AWS Management Console 기본 구조 이해
- 서비스 검색 및 접근 방법 습득
- 리전(Region) 개념 이해 및 선택 방법
- 즐겨찾기 및 최근 방문 서비스 활용
- 도움말 리소스 및 서비스 상태 확인 방법

## ⏱️ 예상 소요 시간

약 15-20분

## 💰 예상 비용

$0.00 (Console 탐색은 무료)

## 1️⃣ AWS Management Console 로그인

### Step 1: Console 접속

1. 웹 브라우저에서 [https://console.aws.amazon.com/](https://console.aws.amazon.com/) 접속
2. **루트 사용자** 또는 **IAM 사용자** 선택
   - 일상 작업: IAM 사용자 권장
   - 계정 관리: 루트 사용자

### Step 2: 자격 증명 입력

#### 루트 사용자 로그인

1. **루트 사용자 이메일 주소** 입력
2. **"다음"** 클릭
3. **암호** 입력
4. **MFA 코드** 입력 (활성화된 경우)
5. **"로그인"** 클릭

#### IAM 사용자 로그인

1. **계정 ID** 또는 **계정 별칭** 입력
2. **"다음"** 클릭
3. **IAM 사용자 이름** 입력
4. **암호** 입력
5. **MFA 코드** 입력 (활성화된 경우)
6. **"로그인"** 클릭

## 2️⃣ Console 기본 구조

### 상단 네비게이션 바

Console 상단에는 다음과 같은 주요 요소가 있습니다:

```
┌─────────────────────────────────────────────────────────────────┐
│ [AWS 로고] [서비스▼] [리소스 그룹▼] [검색창] [리전] [알림] [계정▼] │
└─────────────────────────────────────────────────────────────────┘
```

#### 주요 구성 요소

1. **AWS 로고** (좌측 상단)
   - 클릭 시 Console 홈으로 이동

2. **서비스 메뉴** (Services)
   - 모든 AWS 서비스 카테고리별 목록
   - 최근 방문 서비스 표시

3. **리소스 그룹** (Resource Groups)
   - 관련 리소스를 그룹으로 관리
   - 태그 기반 리소스 필터링

4. **검색창** (Search)
   - 서비스, 기능, 문서 통합 검색
   - 단축키: `/` (슬래시)

5. **리전 선택기** (Region Selector)
   - 현재 선택된 리전 표시
   - 리전 변경 가능

6. **알림 아이콘** (Notifications)
   - AWS 서비스 상태 알림
   - 계정 관련 알림

7. **계정 메뉴** (Account)
   - 계정 정보
   - 결제 및 비용 관리
   - 보안 자격 증명
   - 로그아웃

## 3️⃣ 서비스 검색 및 접근

### 방법 1: 서비스 메뉴 사용

#### Step 1: 서비스 메뉴 열기

1. 상단 네비게이션 바에서 **"서비스"** 클릭
2. 드롭다운 메뉴가 열림

#### Step 2: 카테고리별 탐색

서비스는 다음 카테고리로 분류됩니다:

- **컴퓨팅** (Compute)
  - EC2, Lambda, Elastic Beanstalk 등

- **스토리지** (Storage)
  - S3, EBS, EFS, Glacier 등

- **데이터베이스** (Database)
  - RDS, DynamoDB, ElastiCache 등

- **네트워킹 및 콘텐츠 전송** (Networking & Content Delivery)
  - VPC, CloudFront, Route 53 등

- **보안, 자격 증명 및 규정 준수** (Security, Identity & Compliance)
  - IAM, Cognito, KMS 등

- **관리 및 거버넌스** (Management & Governance)
  - CloudWatch, CloudFormation, Systems Manager 등

#### Step 3: 서비스 선택

1. 원하는 카테고리 클릭
2. 서비스 목록에서 원하는 서비스 클릭
3. 해당 서비스 Console로 이동

### 방법 2: 통합 검색 사용 (권장)

#### Step 1: 검색창 활성화

1. 상단 검색창 클릭
2. 또는 키보드에서 **`/`** (슬래시) 입력

#### Step 2: 검색어 입력

1. 서비스 이름 입력 (예: `EC2`, `S3`, `Lambda`)
2. 기능 이름 입력 (예: `인스턴스 시작`, `버킷 생성`)
3. 자동 완성 제안 표시

#### Step 3: 결과 선택

검색 결과는 다음과 같이 분류됩니다:

- **서비스** (Services): AWS 서비스 목록
- **기능** (Features): 특정 작업 바로가기
- **문서** (Documentation): AWS 공식 문서
- **블로그** (Blogs): AWS 블로그 게시물

### 방법 3: 최근 방문 서비스

#### 최근 방문 서비스 확인

1. **"서비스"** 메뉴 클릭
2. 상단에 **"최근 방문"** 섹션 표시
3. 최근 사용한 서비스 최대 5개 표시
4. 클릭하여 빠르게 접근

## 4️⃣ 리전(Region) 선택

### 리전이란?

- AWS 데이터 센터가 위치한 **지리적 영역**
- 각 리전은 **독립적**으로 운영
- 리소스는 **리전별로 분리**되어 관리

### 리전 선택 중요성

1. **지연 시간(Latency)**: 사용자와 가까운 리전 선택 시 응답 속도 향상
2. **규정 준수(Compliance)**: 데이터 저장 위치 규제 준수
3. **서비스 가용성**: 일부 서비스는 특정 리전에서만 제공
4. **비용**: 리전별로 가격이 다를 수 있음

### 리전 선택 방법

#### Step 1: 리전 선택기 열기

1. 상단 네비게이션 바 우측의 **리전 이름** 클릭
   - 예: "미국 동부(버지니아 북부)" 또는 "US East (N. Virginia)"

#### Step 2: 리전 목록 확인

리전 목록이 표시됩니다:

```
아시아 태평양
  ✓ 서울 (ap-northeast-2)
  • 도쿄 (ap-northeast-1)
  • 싱가포르 (ap-southeast-1)
  • 시드니 (ap-southeast-2)

미국
  • 버지니아 북부 (us-east-1)
  • 오하이오 (us-east-2)
  • 캘리포니아 북부 (us-west-1)
  • 오레곤 (us-west-2)

유럽
  • 아일랜드 (eu-west-1)
  • 프랑크푸르트 (eu-central-1)
  • 런던 (eu-west-2)
```

#### Step 3: 리전 선택

1. 원하는 리전 클릭
2. Console이 선택한 리전으로 전환
3. 해당 리전의 리소스만 표시

### 리전 선택 권장사항

#### 한국 사용자

- **서울 리전 (ap-northeast-2)** 권장
- 가장 낮은 지연 시간
- 한국어 지원 서비스 제공

#### 학습 목적

- **버지니아 북부 (us-east-1)** 권장
- 가장 많은 서비스 제공
- 최신 기능 우선 출시
- 대부분의 튜토리얼 기준 리전

### 글로벌 서비스

일부 서비스는 **리전 독립적**입니다:

- **IAM** (Identity and Access Management)
- **Route 53** (DNS 서비스)
- **CloudFront** (CDN)
- **WAF** (Web Application Firewall)

이러한 서비스는 리전 선택과 무관하게 전역적으로 관리됩니다.

## 5️⃣ 즐겨찾기 및 바로가기

### 즐겨찾기 설정

자주 사용하는 서비스를 즐겨찾기에 추가하여 빠르게 접근할 수 있습니다.

#### Step 1: 서비스 메뉴 열기

1. 상단 **"서비스"** 클릭

#### Step 2: 즐겨찾기 추가

1. 원하는 서비스 위에 마우스 오버
2. 서비스 이름 옆 **별 아이콘(★)** 클릭
3. 별이 채워지면 즐겨찾기 추가 완료

#### Step 3: 즐겨찾기 확인

1. **"서비스"** 메뉴 클릭
2. 상단에 **"즐겨찾기"** 섹션 표시
3. 추가한 서비스 목록 확인

#### Step 4: 즐겨찾기 제거

1. 즐겨찾기에서 제거할 서비스의 **별 아이콘(★)** 다시 클릭
2. 별이 비워지면 제거 완료

### 바로가기 키보드 단축키

효율적인 Console 탐색을 위한 단축키:

| 단축키 | 기능 |
|--------|------|
| `/` | 검색창 활성화 |
| `g` + `h` | Console 홈으로 이동 |
| `g` + `s` | 서비스 메뉴 열기 |
| `Esc` | 현재 메뉴 닫기 |

## 6️⃣ 도움말 리소스 접근

### AWS 도움말 패널

#### Step 1: 도움말 패널 열기

1. 우측 하단의 **물음표 아이콘(?)** 클릭
2. 도움말 패널이 우측에서 슬라이드 인

#### Step 2: 도움말 옵션

도움말 패널에서 다음 리소스에 접근할 수 있습니다:

- **문서** (Documentation)
  - 현재 서비스의 공식 문서
  - 사용자 가이드, API 레퍼런스

- **지원 센터** (Support Center)
  - 지원 케이스 생성 및 관리
  - 기술 지원 요청

- **피드백** (Feedback)
  - Console 개선 제안
  - 버그 리포트

- **바로가기 키** (Keyboard Shortcuts)
  - 사용 가능한 단축키 목록

### AWS 문서 센터

#### 접근 방법

1. [https://docs.aws.amazon.com/](https://docs.aws.amazon.com/) 접속
2. 또는 검색창에서 **"문서"** 검색

#### 주요 문서 유형

- **사용자 가이드** (User Guide)
  - 서비스 개요 및 사용법
  - 단계별 튜토리얼

- **API 레퍼런스** (API Reference)
  - API 호출 방법
  - 파라미터 및 응답 형식

- **CLI 레퍼런스** (CLI Reference)
  - AWS CLI 명령어
  - 옵션 및 예제

- **SDK 레퍼런스** (SDK Reference)
  - 프로그래밍 언어별 SDK
  - 코드 예제

### AWS 교육 리소스

#### AWS Skill Builder

1. [https://skillbuilder.aws/](https://skillbuilder.aws/) 접속
2. 무료 온라인 교육 과정
3. 실습 랩 환경 제공

#### AWS 백서 및 가이드

1. [https://aws.amazon.com/ko/whitepapers/](https://aws.amazon.com/ko/whitepapers/) 접속
2. 아키텍처 모범 사례
3. 보안 및 규정 준수 가이드

## 7️⃣ 서비스 상태 확인

### AWS Service Health Dashboard

#### 전체 서비스 상태 확인

1. [https://status.aws.amazon.com/](https://status.aws.amazon.com/) 접속
2. 모든 리전의 서비스 상태 실시간 확인
3. 과거 장애 이력 조회

#### 상태 표시

- **초록색**: 정상 운영
- **주황색**: 성능 저하
- **빨간색**: 서비스 중단

### Personal Health Dashboard

개인 계정에 영향을 미치는 이벤트를 확인합니다.

#### Step 1: Personal Health Dashboard 접속

1. AWS Console 로그인
2. 상단 검색창에 **"Health"** 입력
3. **"AWS Health"** 서비스 선택

#### Step 2: 이벤트 확인

1. **"대시보드"** 탭에서 현재 이벤트 확인
2. **"이벤트 로그"** 탭에서 과거 이벤트 조회
3. 영향받는 리소스 목록 확인

#### Step 3: 알림 설정

1. **"알림 기본 설정"** 클릭
2. 이메일 또는 SMS 알림 설정
3. EventBridge 규칙으로 자동화 가능

## 8️⃣ Console 설정 커스터마이징

### 언어 설정

#### Step 1: 언어 변경

1. 우측 상단 **계정 메뉴** 클릭
2. **"언어 설정"** 선택
3. 원하는 언어 선택 (한국어, 영어 등)
4. **"저장"** 클릭

> 💡 **참고**: 일부 서비스는 영어로만 제공될 수 있습니다.

### 테마 설정

#### Step 1: 다크 모드 활성화

1. 우측 하단 **설정 아이콘(⚙️)** 클릭
2. **"다크 모드"** 토글 활성화
3. Console이 다크 테마로 전환

### 홈 화면 커스터마이징

#### Step 1: 위젯 추가/제거

1. Console 홈 화면 접속
2. **"위젯 추가"** 클릭
3. 원하는 위젯 선택:
   - 비용 및 사용량
   - 최근 방문 서비스
   - 리소스 그룹
   - 서비스 상태
4. **"추가"** 클릭

#### Step 2: 위젯 배치

1. 위젯 상단을 드래그하여 위치 이동
2. 위젯 크기 조절 (일부 위젯)
3. 불필요한 위젯 **"X"** 버튼으로 제거

## ❓ 문제 해결

### Console이 느리게 로드되는 경우

1. 브라우저 캐시 및 쿠키 삭제
2. 다른 브라우저 시도 (Chrome, Firefox, Edge 권장)
3. 브라우저 확장 프로그램 비활성화
4. 인터넷 연결 상태 확인

### 리전 변경 후 리소스가 보이지 않는 경우

- 리소스는 **리전별로 분리**되어 있습니다
- 리소스를 생성한 리전으로 다시 전환하세요
- 글로벌 서비스(IAM, Route 53 등)는 리전 무관

### 서비스 메뉴에서 원하는 서비스가 보이지 않는 경우

1. 검색 기능 사용 (단축키: `/`)
2. 해당 서비스가 선택한 리전에서 제공되는지 확인
3. 계정에 해당 서비스 사용 권한이 있는지 확인

### MFA 코드 입력 후 로그인 실패

1. 스마트폰 시간이 자동 동기화되어 있는지 확인
2. 인증 앱을 최신 버전으로 업데이트
3. 새로운 MFA 코드로 재시도 (30초 대기)

## 💡 유용한 팁

### 효율적인 Console 사용

1. **검색 기능 적극 활용**
   - 서비스 이름 일부만 입력해도 검색 가능
   - 예: `ec2`, `s3`, `lam` (Lambda)

2. **즐겨찾기 활용**
   - 자주 사용하는 서비스 5-10개 추가
   - 프로젝트별로 관련 서비스 그룹화

3. **리전 고정**
   - 학습 중에는 하나의 리전에서 작업 권장
   - 리소스 관리 용이

4. **브라우저 북마크**
   - 자주 사용하는 서비스 Console URL 북마크
   - 예: `https://console.aws.amazon.com/ec2/`

5. **여러 탭 활용**
   - 여러 서비스를 동시에 작업할 때 유용
   - 탭별로 다른 리전 선택 가능
