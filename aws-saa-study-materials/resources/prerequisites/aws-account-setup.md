# AWS 계정 설정 가이드

## 📋 개요

이 가이드는 AWS 학습을 시작하기 위한 AWS 계정 설정 과정을 안내합니다. AWS Free Tier를 활용하여 비용 부담 없이 학습할 수 있도록 계정을 설정하고, 예상치 못한 비용 발생을 방지하기 위한 보안 설정을 진행합니다.

## 🎯 학습 목표

- AWS Free Tier 계정 생성
- 결제 알람 설정으로 비용 관리
- 루트 계정 보안 강화 (MFA 활성화)
- AWS 계정 보안 베스트 프랙티스 이해

## ⏱️ 예상 소요 시간

약 20-30분

## 💰 예상 비용

$0.00 (계정 생성 및 설정은 무료)

## 1️⃣ AWS Free Tier 계정 생성

### 필요한 정보

계정 생성을 위해 다음 정보를 준비하세요:

- ✉️ 유효한 이메일 주소
- 📱 휴대폰 번호 (SMS 인증용)
- 💳 신용카드 또는 체크카드 (본인 확인용, $1 임시 승인 후 취소됨)
- 🏠 청구지 주소

### 단계별 가이드

#### Step 1: AWS 홈페이지 접속

1. 웹 브라우저에서 [https://aws.amazon.com/ko/](https://aws.amazon.com/ko/) 접속
2. 우측 상단의 **"AWS 계정 생성"** 버튼 클릭

#### Step 2: 계정 정보 입력

1. **이메일 주소** 입력
   - 향후 AWS 알림을 받을 이메일 주소
   - 회사 이메일 또는 개인 이메일 사용 가능

2. **AWS 계정 이름** 입력
   - 예: "MyAWS-Learning-Account"
   - 나중에 변경 가능

3. **"이메일 주소 확인"** 클릭
   - 입력한 이메일로 인증 코드 전송됨
   - 이메일 확인 후 인증 코드 입력

#### Step 3: 루트 사용자 암호 설정

1. **강력한 암호** 생성
   - 최소 8자 이상
   - 대문자, 소문자, 숫자, 특수문자 포함 권장
   - 예: `MyAWS@2024!Secure`

2. **암호 확인** 재입력

3. **"계속"** 버튼 클릭

> ⚠️ **중요**: 이 암호는 루트 계정 암호입니다. 안전하게 보관하세요!

#### Step 4: 연락처 정보 입력

1. **계정 유형** 선택
   - **개인**: 개인 학습용 (권장)
   - **비즈니스**: 회사 업무용

2. **연락처 정보** 입력
   - 이름
   - 전화번호
   - 국가/지역: 대한민국
   - 주소

3. **AWS 고객 계약** 동의 체크

4. **"계속"** 버튼 클릭

#### Step 5: 결제 정보 입력

1. **신용카드 또는 체크카드** 정보 입력
   - 카드 번호
   - 만료일
   - 카드 소유자 이름
   - 청구지 주소

2. **"보안 확인 및 계속"** 클릭

> 💡 **참고**: AWS는 본인 확인을 위해 $1를 임시로 승인하지만, 실제로 청구되지 않고 곧 취소됩니다.

#### Step 6: 자격 증명 확인

1. **전화번호 인증** 선택
   - 문자 메시지(SMS) 또는 음성 통화 선택

2. **보안 확인 코드** 입력
   - CAPTCHA 문자 입력

3. **"SMS 전송"** 또는 **"전화 걸기"** 클릭

4. **인증 코드** 입력
   - SMS 또는 음성으로 받은 4자리 코드 입력

5. **"계속"** 클릭

#### Step 7: 지원 플랜 선택

1. **기본 지원 - 무료** 선택 (권장)
   - 학습 목적으로는 기본 플랜으로 충분
   - 나중에 필요시 업그레이드 가능

2. **"가입 완료"** 클릭

#### Step 8: 계정 활성화 대기

- 계정 활성화까지 **최대 24시간** 소요 (보통 몇 분 내 완료)
- 활성화 완료 시 이메일 알림 수신
- 활성화 후 AWS Management Console 로그인 가능

## 2️⃣ 결제 알람 설정

예상치 못한 비용 발생을 방지하기 위해 결제 알람을 설정합니다.

### Step 1: Billing Dashboard 접속

1. [AWS Management Console](https://console.aws.amazon.com/) 로그인
2. 우측 상단 **계정 이름** 클릭
3. **"결제 및 비용 관리"** 선택

### Step 2: 결제 기본 설정

1. 좌측 메뉴에서 **"결제 기본 설정"** 클릭
2. **"결제 알림 수신"** 섹션에서:
   - ✅ **"PDF 송장을 이메일로 수신"** 체크
   - ✅ **"무료 사용량 알림 수신"** 체크
   - ✅ **"결제 알림 수신"** 체크

3. **알림 이메일 주소** 확인
4. **"기본 설정 업데이트"** 클릭

### Step 3: CloudWatch 결제 알람 생성

#### 3-1. CloudWatch 서비스 접속

1. 상단 검색창에 **"CloudWatch"** 입력
2. **CloudWatch** 서비스 선택
3. 리전을 **"미국 동부(버지니아 북부) us-east-1"** 로 변경
   - 결제 메트릭은 us-east-1 리전에서만 사용 가능

#### 3-2. 알람 생성

1. 좌측 메뉴에서 **"경보"** > **"모든 경보"** 클릭
2. **"경보 생성"** 버튼 클릭

#### 3-3. 메트릭 선택

1. **"메트릭 선택"** 클릭
2. **"결제"** 카테고리 선택
3. **"총 예상 요금"** 선택
4. **"통화: USD"** 체크
5. **"메트릭 선택"** 클릭

#### 3-4. 조건 설정

1. **임계값 유형**: 정적
2. **다음 시기**: 보다 큼
3. **임계값**: `5` (USD) 입력
   - 권장: $5 ~ $10 (Free Tier 초과 시 알림)
   - 필요에 따라 조정 가능

4. **"다음"** 클릭

#### 3-5. 알림 설정

1. **"새 주제 생성"** 선택
2. **주제 이름**: `billing-alarm-topic` 입력
3. **알림을 받을 이메일 주소** 입력
4. **"주제 생성"** 클릭
5. **"다음"** 클릭

> 📧 **중요**: 입력한 이메일로 **"AWS Notification - Subscription Confirmation"** 이메일이 발송됩니다. 이메일을 열고 **"Confirm subscription"** 링크를 클릭해야 알람이 활성화됩니다!

#### 3-6. 알람 이름 설정

1. **경보 이름**: `BillingAlarm-5USD` 입력
2. **경보 설명**: `Alert when estimated charges exceed $5` 입력
3. **"다음"** 클릭

#### 3-7. 검토 및 생성

1. 설정 내용 확인
2. **"경보 생성"** 클릭

### Step 4: 알람 확인

1. 이메일 받은편지함 확인
2. **"AWS Notification - Subscription Confirmation"** 이메일 열기
3. **"Confirm subscription"** 링크 클릭
4. 브라우저에서 "Subscription confirmed!" 메시지 확인

## 3️⃣ 루트 계정 보안 강화 (MFA 활성화)

Multi-Factor Authentication(MFA)을 활성화하여 계정 보안을 강화합니다.

### MFA란?

- **2단계 인증** 방식
- 암호 + 인증 코드(OTP) 조합으로 로그인
- 계정 해킹 위험 대폭 감소

### 필요한 준비물

다음 중 하나를 선택:

1. **스마트폰 인증 앱** (권장)
   - Google Authenticator (Android/iOS)
   - Microsoft Authenticator (Android/iOS)
   - Authy (Android/iOS)

2. **하드웨어 MFA 디바이스** (선택사항)
   - YubiKey 등

### Step 1: IAM 대시보드 접속

1. AWS Management Console 로그인
2. 상단 검색창에 **"IAM"** 입력
3. **IAM** 서비스 선택

### Step 2: 루트 사용자 MFA 활성화

1. 우측 **"보안 권장 사항"** 패널에서:
   - **"루트 사용자에 대해 MFA 추가"** 클릭

2. 또는 좌측 메뉴에서:
   - **"대시보드"** 클릭
   - **"보안 권장 사항"** 섹션에서 **"MFA 추가"** 클릭

### Step 3: MFA 디바이스 선택

1. **"MFA 디바이스 할당"** 버튼 클릭
2. **디바이스 이름** 입력: `root-mfa-device`
3. **MFA 디바이스 선택**:
   - ✅ **"인증 관리자 앱"** 선택 (권장)
4. **"다음"** 클릭

### Step 4: 인증 앱 설정

#### 4-1. 인증 앱 설치 (아직 없는 경우)

- **Android**: Google Play Store에서 "Google Authenticator" 검색 및 설치
- **iOS**: App Store에서 "Google Authenticator" 검색 및 설치

#### 4-2. QR 코드 스캔

1. AWS Console에 표시된 **QR 코드**를 인증 앱으로 스캔
   - Google Authenticator 앱 열기
   - **"+"** 버튼 클릭
   - **"QR 코드 스캔"** 선택
   - AWS Console의 QR 코드를 카메라로 스캔

2. 또는 **"비밀 키 표시"** 클릭하여 수동 입력
   - 인증 앱에서 **"설정 키 입력"** 선택
   - 계정 이름: `AWS-Root`
   - 키: AWS Console에 표시된 비밀 키 입력

#### 4-3. MFA 코드 입력

1. 인증 앱에 표시된 **6자리 코드** 확인
2. AWS Console의 **"MFA 코드 1"** 에 입력
3. 약 30초 대기 후 새로운 코드 생성되면
4. **"MFA 코드 2"** 에 새 코드 입력
5. **"MFA 추가"** 클릭

### Step 5: MFA 활성화 확인

1. **"MFA가 성공적으로 할당되었습니다"** 메시지 확인
2. IAM 대시보드에서 **"보안 권장 사항"** 확인
   - ✅ "루트 사용자에 대해 MFA 추가" 항목이 완료 상태로 표시

## 4️⃣ 루트 계정 보안 베스트 프랙티스

### ✅ 해야 할 것

1. **MFA 활성화** (위에서 완료)
2. **강력한 암호 사용**
   - 최소 12자 이상
   - 대소문자, 숫자, 특수문자 조합
3. **루트 계정 액세스 키 생성하지 않기**
   - 프로그래밍 방식 액세스에는 IAM 사용자 사용
4. **루트 계정 사용 최소화**
   - 일상적인 작업에는 IAM 사용자 사용
   - 루트 계정은 계정 관리 작업에만 사용

### ❌ 하지 말아야 할 것

1. **루트 계정으로 일상 작업 수행**
2. **루트 계정 자격 증명 공유**
3. **루트 계정 액세스 키 생성**
4. **MFA 없이 루트 계정 사용**

### 🔐 추가 보안 권장사항

1. **CloudTrail 활성화**
   - 모든 API 호출 로깅
   - 보안 감사 및 규정 준수

2. **정기적인 보안 검토**
   - IAM 자격 증명 보고서 확인
   - 사용하지 않는 자격 증명 제거

3. **결제 알람 모니터링**
   - 정기적으로 비용 확인
   - 예상치 못한 요금 발생 시 즉시 조사

## ❓ 문제 해결

### 계정 활성화가 24시간 이상 지연되는 경우

- AWS Support에 문의: [https://console.aws.amazon.com/support/home](https://console.aws.amazon.com/support/home)
- 전화: 1-877-742-2121 (미국 무료 전화)

### 결제 알람 이메일이 오지 않는 경우

1. 스팸 폴더 확인
2. 이메일 주소가 올바른지 확인
3. SNS 구독 상태 확인: CloudWatch > 경보 > 알림 탭

### MFA 코드가 작동하지 않는 경우

1. 스마트폰 시간이 자동 동기화되어 있는지 확인
2. 인증 앱을 최신 버전으로 업데이트
3. QR 코드를 다시 스캔하여 재설정
