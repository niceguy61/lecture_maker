## Day 1: AWS 계정 설정 및 기본 환경 구성

### 🎯 목표
AWS 계정 생성 및 기본 보안 설정을 통한 안전한 학습 환경 구축

### ✅ 실습 체크리스트
- [ ] **AWS 계정 생성**
  - 이메일 주소 및 결제 정보 등록
  - 계정 활성화 완료
  - 루트 사용자 로그인 확인

- [ ] **루트 사용자 보안 강화**
  - MFA(Multi-Factor Authentication) 설정
  - 강력한 비밀번호 설정
  - 보안 질문 설정

- [ ] **결제 및 비용 관리**
  - 결제 알림 설정 (Free Tier 초과 시)
  - 예산 알림 설정 ($5-10 수준)
  - Cost Explorer 접근 확인

- [ ] **기본 환경 설정**
  - 기본 리전 선택 (us-east-1 권장)
  - AWS CLI 설치 (로컬 환경)
  - AWS CLI 기본 구성 확인

### 📝 완료 확인 방법
- AWS Console 정상 로그인
- MFA 인증 정상 작동
- 결제 알림 이메일 수신 설정 확인

## Day 2: IAM (Identity and Access Management)

### 🎯 목표
IAM을 통한 사용자, 그룹, 역할 관리 및 최소 권한 원칙 적용

### ✅ 실습 체크리스트
- [ ] **IAM 사용자 생성**
  - 관리자 권한 사용자 생성
  - 프로그래밍 방식 액세스 활성화
  - 콘솔 액세스 활성화

- [ ] **IAM 그룹 관리**
  - 개발자 그룹 생성
  - AdministratorAccess 정책 연결
  - 사용자를 그룹에 추가

- [ ] **IAM 역할 생성**
  - EC2 인스턴스용 역할 생성
  - AmazonSSMManagedInstanceCore 정책 연결
  - CloudWatchAgentServerPolicy 정책 연결

- [ ] **보안 강화**
  - IAM 사용자 MFA 설정
  - 액세스 키 생성 및 안전한 저장
  - 비밀번호 정책 설정

- [ ] **권한 테스트**
  - IAM 사용자로 로그인 테스트
  - EC2 서비스 접근 권한 확인
  - 루트 사용자 사용 중단

### 📝 완료 확인 방법
- IAM 사용자로 정상 로그인
- EC2 콘솔 접근 가능
- 역할 생성 및 정책 연결 확인

## Day 3: EC2 (Elastic Compute Cloud) 기초

### 🎯 목표
EC2 인스턴스 생성, 관리 및 기본 웹 서버 구성

### ✅ 실습 체크리스트
- [ ] **키 페어 관리**
  - EC2 키 페어 생성
  - 프라이빗 키 안전한 위치에 저장
  - 키 파일 권한 설정 (chmod 400)

- [ ] **보안 그룹 생성**
  - 웹 서버용 보안 그룹 생성
  - HTTP(80), HTTPS(443) 포트 개방
  - SSH(22) 포트 개방 (본인 IP만)

- [ ] **EC2 인스턴스 시작**
  - Amazon Linux 2023 AMI 선택
  - t2.micro 인스턴스 유형 선택
  - 기본 VPC 및 퍼블릭 서브넷 선택
  - 퍼블릭 IP 자동 할당 활성화

- [ ] **인스턴스 연결**
  - SSH를 통한 인스턴스 접속
  - 시스템 업데이트 실행
  - 기본 명령어 실행 테스트

- [ ] **웹 서버 설정**
  - Apache 또는 Nginx 설치
  - 웹 서버 시작 및 자동 시작 설정
  - 기본 HTML 페이지 생성
  - 웹 브라우저를 통한 접근 테스트

### 📝 완료 확인 방법
- SSH 연결 성공
- 웹 브라우저에서 퍼블릭 IP로 접근 가능
- 사용자 정의 HTML 페이지 표시

## Day 4: EC2 고급 기능 및 스토리지

### 🎯 목표
EBS 볼륨 관리, 스냅샷 생성 및 인스턴스 고급 기능 활용

### ✅ 실습 체크리스트
- [ ] **EBS 볼륨 생성**
  - 추가 EBS 볼륨 생성 (8GB, gp3)
  - 인스턴스와 동일한 AZ에 생성
  - 볼륨을 인스턴스에 연결

- [ ] **파일 시스템 구성**
  - 새 볼륨 파티션 생성
  - 파일 시스템 포맷 (ext4)
  - 마운트 포인트 생성 및 마운트
  - /etc/fstab 설정으로 영구 마운트

- [ ] **스냅샷 관리**
  - 루트 볼륨 스냅샷 생성
  - 추가 볼륨 스냅샷 생성
  - 스냅샷 설명 및 태그 추가

- [ ] **인스턴스 관리**
  - 인스턴스 중지 및 시작
  - 인스턴스 유형 변경 (t2.micro → t2.small → t2.micro)
  - 사용자 데이터 스크립트 활용

- [ ] **모니터링 설정**
  - CloudWatch 모니터링 활성화
  - 기본 메트릭 확인
  - 인스턴스 상태 확인

### 📝 완료 확인 방법
- 추가 볼륨 정상 마운트 및 사용 가능
- 스냅샷 생성 완료
- 인스턴스 유형 변경 성공

## Day 5: VPC (Virtual Private Cloud) 기초

### 🎯 목표
사용자 정의 VPC 생성 및 기본 네트워킹 구성

### ✅ 실습 체크리스트
- [ ] **VPC 생성**
  - 사용자 정의 VPC 생성 (10.0.0.0/16)
  - DNS 호스트 이름 활성화
  - DNS 확인 활성화

- [ ] **서브넷 생성**
  - 퍼블릭 서브넷 생성 (10.0.1.0/24)
  - 프라이빗 서브넷 생성 (10.0.2.0/24)
  - 서로 다른 AZ에 배치

- [ ] **인터넷 게이트웨이**
  - 인터넷 게이트웨이 생성
  - VPC에 연결
  - 상태 확인

- [ ] **라우팅 테이블 구성**
  - 퍼블릭 라우팅 테이블 생성
  - 인터넷 게이트웨이로의 기본 라우트 추가
  - 퍼블릭 서브넷 연결

- [ ] **EC2 인스턴스 배포**
  - 사용자 정의 VPC에 인스턴스 배포
  - 퍼블릭 서브넷에 웹 서버 배치
  - 프라이빗 서브넷에 앱 서버 배치

### 📝 완료 확인 방법
- VPC 내 인스턴스 정상 생성
- 퍼블릭 서브넷 인스턴스 인터넷 접근 가능
- 프라이빗 서브넷 인스턴스 인터넷 직접 접근 불가

## Day 6: VPC 고급 네트워킹

### 🎯 목표
NAT Gateway, VPC Endpoint 구성 및 고급 보안 설정

### ✅ 실습 체크리스트
- [ ] **NAT Gateway 구성**
  - Elastic IP 할당
  - 퍼블릭 서브넷에 NAT Gateway 생성
  - 프라이빗 라우팅 테이블에 NAT Gateway 라우트 추가

- [ ] **VPC Endpoint 생성**
  - S3용 Gateway Endpoint 생성
  - 라우팅 테이블에 자동 라우트 추가 확인
  - 프라이빗 서브넷에서 S3 접근 테스트

- [ ] **NACL 구성**
  - 사용자 정의 NACL 생성
  - 웹 트래픽 허용 규칙 추가
  - 서브넷에 NACL 연결

- [ ] **보안 그룹 고급 설정**
  - 계층별 보안 그룹 생성
  - 보안 그룹 간 참조 규칙 설정
  - 최소 권한 원칙 적용

- [ ] **연결성 테스트**
  - 프라이빗 인스턴스 인터넷 아웃바운드 접근 확인
  - VPC Endpoint를 통한 S3 접근 확인
  - 보안 그룹 규칙 동작 검증

### 📝 완료 확인 방법
- 프라이빗 인스턴스에서 인터넷 접근 가능 (NAT Gateway 경유)
- S3 VPC Endpoint 정상 작동
- 보안 그룹 규칙 정상 적용

## Day 7: 종합 실습 - 3-Tier 아키텍처

### 🎯 목표
Week 1 학습 내용을 종합한 3-tier 웹 애플리케이션 아키텍처 구축

### ✅ 실습 체크리스트

#### VPC 및 네트워크 구성
- [ ] **VPC 생성**
  - MyApp-VPC 생성 (10.0.0.0/16)
  - DNS 설정 활성화

- [ ] **서브넷 생성 (6개)**
  - 퍼블릭 서브넷 2개 (Multi-AZ)
  - 프라이빗 앱 서브넷 2개 (Multi-AZ)
  - 프라이빗 DB 서브넷 2개 (Multi-AZ)

- [ ] **게이트웨이 구성**
  - 인터넷 게이트웨이 생성 및 연결
  - NAT Gateway 생성 (Elastic IP 포함)

- [ ] **라우팅 테이블 구성**
  - 퍼블릭 라우팅 테이블
  - 프라이빗 앱 라우팅 테이블
  - 프라이빗 DB 라우팅 테이블

#### 보안 구성
- [ ] **보안 그룹 생성 (3개)**
  - 웹 서버 보안 그룹 (HTTP, HTTPS, SSH)
  - 앱 서버 보안 그룹 (웹 서버에서만 접근)
  - DB 서버 보안 그룹 (앱 서버에서만 접근)

- [ ] **IAM 역할 생성**
  - EC2 인스턴스용 역할
  - 필요한 정책 연결

#### 인스턴스 배포
- [ ] **웹 서버 인스턴스**
  - 퍼블릭 서브넷에 배치
  - Apache/Nginx 설치
  - 사용자 데이터 스크립트 활용

- [ ] **앱 서버 인스턴스**
  - 프라이빗 앱 서브넷에 배치
  - Java/Python 런타임 설치
  - 웹 서버에서 접근 가능

- [ ] **DB 서버 인스턴스**
  - 프라이빗 DB 서브넷에 배치
  - MySQL/MariaDB 설치
  - 앱 서버에서만 접근 가능

#### 연결성 및 보안 테스트
- [ ] **외부 접근 테스트**
  - 웹 서버 퍼블릭 IP로 HTTP 접근
  - 웹 페이지 정상 표시 확인

- [ ] **내부 연결 테스트**
  - 웹 서버 → 앱 서버 연결
  - 앱 서버 → DB 서버 연결
  - 각 티어 간 통신 확인

- [ ] **보안 검증**
  - 앱/DB 서버 직접 인터넷 접근 차단 확인
  - 불필요한 포트 차단 확인
  - 보안 그룹 규칙 정상 작동 확인

### 📝 완료 확인 방법
- 웹 브라우저에서 웹 서버 접근 가능
- 모든 티어 간 내부 통신 정상
- 보안 규칙 정상 적용
- 아키텍처 다이어그램과 일치
