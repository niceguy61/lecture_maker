## 문제 1 (AWS 글로벌 인프라)
**난이도**: ⭐⭐☆

회사에서 글로벌 사용자를 대상으로 하는 웹 애플리케이션을 배포하려고 합니다. 사용자들이 가장 빠른 응답 시간을 경험할 수 있도록 하려면 어떤 AWS 구성 요소를 활용해야 합니까?

A) 여러 리전에 애플리케이션 배포  
B) 단일 리전 내 여러 가용 영역 활용  
C) CloudFront와 엣지 로케이션 활용  
D) 더 큰 인스턴스 유형 사용  

**정답**: C) CloudFront와 엣지 로케이션 활용

**해설**: CloudFront는 AWS의 CDN(Content Delivery Network) 서비스로, 전 세계에 분산된 엣지 로케이션을 통해 사용자에게 가장 가까운 위치에서 콘텐츠를 제공합니다. 이를 통해 지연 시간을 최소화하고 사용자 경험을 향상시킬 수 있습니다.

## 문제 2 (AWS 글로벌 인프라)
**난이도**: ⭐⭐☆

다음 중 AWS 리전 선택 시 고려해야 할 주요 요소가 아닌 것은?

A) 지연 시간 (Latency)  
B) 규정 준수 (Compliance)  
C) 서비스 가용성  
D) 인스턴스 패밀리 종류  

**정답**: D) 인스턴스 패밀리 종류

**해설**: 인스턴스 패밀리는 대부분의 리전에서 동일하게 제공되므로 리전 선택의 주요 고려사항이 아닙니다. 리전 선택 시에는 지연 시간, 규정 준수 요구사항, 필요한 서비스의 가용성, 비용 등을 고려해야 합니다.

개발팀에서 S3 버킷에 대한 읽기 전용 접근 권한이 필요합니다. 최소 권한 원칙을 적용하여 IAM 정책을 설계할 때 가장 적절한 접근 방법은?

A) 개발자들에게 AdministratorAccess 정책 부여

## 문제 4 (IAM)
**난이도**: ⭐⭐☆

다음 중 IAM 역할(Role)의 주요 특징으로 올바른 것은?

A) 영구적인 자격 증명을 제공한다  
B) 임시 자격 증명을 제공한다  
C) 사용자만 역할을 맡을 수 있다  
D) 역할은 MFA를 지원하지 않는다  

**정답**: B) 임시 자격 증명을 제공한다

**해설**: IAM 역할은 임시 자격 증명을 제공하는 것이 주요 특징입니다. 역할을 맡으면 임시 보안 자격 증명(액세스 키, 시크릿 키, 세션 토큰)을 받게 되며, 이는 일정 시간 후 만료됩니다. 사용자뿐만 아니라 AWS 서비스, 외부 시스템 등도 역할을 맡을 수 있습니다.

## 문제 5 (EC2)
**난이도**: ⭐⭐⭐

웹 애플리케이션을 위한 EC2 인스턴스를 선택할 때, CPU 집약적인 작업보다는 메모리 집약적인 작업이 주를 이룬다면 어떤 인스턴스 패밀리를 선택하는 것이 가장 적절합니까?

A) C5 (Compute Optimized)  
B) M5 (General Purpose)  
C) R5 (Memory Optimized)  
D) T3 (Burstable Performance)  

**정답**: C) R5 (Memory Optimized)

**해설**: R5 인스턴스 패밀리는 메모리 최적화 인스턴스로, 메모리 집약적인 애플리케이션에 최적화되어 있습니다. 인메모리 데이터베이스, 빅데이터 처리, 고성능 컴퓨팅 등의 워크로드에 적합합니다.

다음 중 EBS 볼륨과 인스턴스 스토어의 차이점에 대한 설명으로 올바른 것은?

A) EBS 볼륨은 인스턴스 종료 시 데이터가 삭제된다  
B) 인스턴스 스토어는 네트워크를 통해 연결된다

## 문제 7 (VPC 기초)
**난이도**: ⭐⭐☆

VPC에서 10.0.0.0/16 CIDR 블록을 사용할 때, 이론적으로 생성할 수 있는 최대 IP 주소 개수는?

A) 256개  
B) 1,024개  
C) 65,536개  
D) 16,777,216개  

**정답**: C) 65,536개

**해설**: /16 CIDR 블록은 32 - 16 = 16비트를 호스트 주소로 사용합니다. 2^16 = 65,536개의 IP 주소를 제공합니다. 단, AWS에서는 각 서브넷마다 5개의 IP 주소(네트워크, 브로드캐스트, AWS 예약 주소 3개)를 예약하므로 실제 사용 가능한 주소는 더 적습니다.

A) 웹 서버와 데이터베이스 서버 모두 퍼블릭 서브넷에 배치  
B) 웹 서버와 데이터베이스 서버 모두 프라이빗 서브넷에 배치  
C) 웹 서버는 퍼블릭 서브넷, 데이터베이스 서버는 프라이빗 서브넷에 배치

## 문제 9 (보안 그룹)

보안 그룹에 대한 설명 중 올바른 것은?

A) 보안 그룹은 상태 비저장(Stateless) 방화벽이다  
B) 보안 그룹은 기본적으로 모든 트래픽을 허용한다

## 문제 10 (NAT Gateway)
**난이도**: ⭐⭐⭐

프라이빗 서브넷의 인스턴스가 인터넷에 접근해야 하는 상황에서 NAT Gateway를 사용할 때, NAT Gateway는 어디에 배치되어야 합니까?

A) 프라이빗 서브넷  
B) 퍼블릭 서브넷  
C) 인터넷 게이트웨이  
D) VPC 외부  

**정답**: B) 퍼블릭 서브넷

**해설**: NAT Gateway는 프라이빗 서브넷의 인스턴스가 인터넷에 아웃바운드 연결을 할 수 있도록 하는 관리형 서비스입니다. NAT Gateway 자체가 인터넷에 접근할 수 있어야 하므로 퍼블릭 서브넷에 배치되어야 하며, Elastic IP가 할당되어야 합니다.

## 문제 11 (VPC Endpoint)
**난이도**: ⭐⭐⭐

다음 중 VPC Endpoint를 사용하는 주요 이유는 무엇입니까?

A) 인터넷 속도를 향상시키기 위해  
B) AWS 서비스에 프라이빗하게 접근하기 위해  
C) 보안 그룹 규칙을 단순화하기 위해  
D) 인스턴스 성능을 향상시키기 위해  

**정답**: B) AWS 서비스에 프라이빗하게 접근하기 위해

**해설**: VPC Endpoint는 VPC 내의 인스턴스가 인터넷을 거치지 않고 AWS 서비스(S3, DynamoDB 등)에 프라이빗하게 접근할 수 있도록 하는 서비스입니다. 이를 통해 보안을 강화하고 데이터 전송 비용을 절약할 수 있습니다.

## 문제 12 (통합 아키텍처)
**난이도**: ⭐⭐⭐

3-tier 아키텍처에서 각 티어 간의 통신을 제어하기 위한 가장 적절한 보안 구성은?

A) 모든 티어에 동일한 보안 그룹 적용  
B) 각 티어별로 별도의 보안 그룹을 생성하고, 필요한 포트만 개방  
C) NACL만 사용하여 트래픽 제어  
D) 보안 그룹 없이 서브넷 분리만으로 보안 구현  

**정답**: B) 각 티어별로 별도의 보안 그룹을 생성하고, 필요한 포트만 개방

**해설**: 3-tier 아키텍처에서는 각 티어(웹, 앱, DB)별로 별도의 보안 그룹을 생성하고, 티어 간 통신에 필요한 포트만 선택적으로 개방하는 것이 보안 모범 사례입니다. 예를 들어, 웹 티어는 80/443 포트를 인터넷에 개방하고, 앱 티어는 웹 티어에서만 접근 가능하도록 설정합니다.

## 문제 13 (고가용성)

고가용성을 위해 다중 AZ에 걸쳐 애플리케이션을 배포할 때 고려해야 할 주요 사항은?

- 웹 서버는 인터넷에서 접근 가능
- 애플리케이션 서버는 웹 서버에서만 접근 가능

이 요구사항을 만족하는 가장 적절한 아키텍처는?

A) 모든 서버를 퍼블릭 서브넷에 배치

D) VPC CIDR 블록이 잘못됨 - VPC 재생성 필요  

**정답**: B) 보안 그룹에서 필요한 포트가 차단됨 - 보안 그룹 규칙 수정 필요

**해설**: 같은 VPC 내의 서브넷 간 통신이 되지 않는 가장 일반적인 원인은 보안 그룹 설정입니다. 앱 서버의 보안 그룹에서 웹 서버로부터의 트래픽을 허용하는 인바운드 규칙이 없거나, 웹 서버의 보안 그룹에서 앱 서버로의 아웃바운드 규칙이 없을 수 있습니다.
