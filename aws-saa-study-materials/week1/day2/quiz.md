## 문제 1: IAM 기본 개념 (객관식)

**AWS IAM에 대한 설명으로 올바른 것은?**

A) IAM은 특정 AWS 리전에서만 작동하는 서비스이다  
B) IAM 사용에는 별도의 요금이 부과된다  
C) IAM은 AWS 리소스에 대한 액세스를 안전하게 제어하는 글로벌 서비스이다  
D) IAM 정책은 XML 형식으로만 작성할 수 있다  

<details>
<summary>정답 및 해설</summary>

**정답: C**

**해설:**
- **C (정답)**: IAM은 AWS 리소스에 대한 액세스를 안전하게 제어하는 글로벌 서비스입니다. 모든 AWS 리전에서 동일하게 작동하며, 사용자, 그룹, 역할, 정책을 통해 "누가 무엇을 할 수 있는지"를 관리합니다.

- **A (오답)**: IAM은 글로벌 서비스로, 모든 AWS 리전에서 작동합니다.

- **B (오답)**: IAM 자체 사용에는 추가 비용이 없습니다. 사용자, 그룹, 역할, 정책 생성 및 관리는 무료입니다.

- **D (오답)**: IAM 정책은 JSON 형식으로 작성됩니다.

**관련 개념**: IAM 글로벌 특성, 비용 구조, 정책 형식
</details>

## 문제 2: IAM 구성 요소 (시나리오 기반)

**신입 개발자 김철수가 입사했습니다. 김철수는 개발팀에 속하며, EC2와 S3에 대한 읽기 권한이 필요합니다. 가장 적절한 IAM 구성 방법은?**

A) 김철수에게 직접 EC2와 S3 정책을 개별적으로 연결한다  
B) 'Developers' 그룹을 생성하고 필요한 정책을 연결한 후, 김철수를 이 그룹에 추가한다  
C) 김철수에게 AdministratorAccess 정책을 부여한다  
D) EC2 역할을 생성하여 김철수에게 할당한다  

<details>
<summary>정답 및 해설</summary>

**정답: B**

**해설:**
- **B (정답)**: 그룹을 통한 권한 관리는 IAM의 모범 사례입니다. 'Developers' 그룹에 필요한 정책을 연결하고 사용자를 그룹에 추가하면, 향후 다른 개발자가 입사할 때도 동일한 그룹에 추가하기만 하면 됩니다. 이는 관리가 효율적이고 일관성을 보장합니다.

- **A (오답)**: 개별 사용자에게 직접 정책을 연결하는 것은 관리가 복잡하고 일관성을 보장하기 어렵습니다.

- **C (오답)**: AdministratorAccess는 과도한 권한입니다. 최소 권한 원칙에 위배됩니다.

- **D (오답)**: 역할(Role)은 주로 AWS 서비스나 애플리케이션이 사용하며, 사람(사용자)에게는 직접 할당하지 않습니다.

**관련 개념**: 그룹 기반 권한 관리, 최소 권한 원칙, IAM 모범 사례
</details>

## 문제 3: IAM 정책 구조 (기술적 이해)

**다음 IAM 정책에서 빈 칸에 들어갈 올바른 값은?**

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "______",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-bucket/*",
      "Condition": {
        "StringEquals": {
          "s3:x-amz-server-side-encryption": "AES256"
        }
      }
    }
  ]
}
```

A) Grant  
B) Allow  
C) Permit  
D) Accept  

<details>
<summary>정답 및 해설</summary>

**정답: B**

**해설:**
- **B (정답)**: IAM 정책에서 Effect 필드는 "Allow" 또는 "Deny" 값만 가질 수 있습니다. "Allow"는 지정된 작업을 허용한다는 의미입니다.

- **A, C, D (오답)**: 이들은 IAM 정책에서 사용되지 않는 값들입니다.

**정책 해석**: 이 정책은 "my-bucket" S3 버킷의 객체에 대해 GetObject 작업을 허용하되, 서버 측 암호화가 AES256으로 설정된 경우에만 허용합니다.

**관련 개념**: IAM 정책 구조, Effect 필드, 조건부 정책
</details>

## 문제 4: IAM 역할 vs 사용자 (개념 구분)

**다음 중 IAM 역할(Role)의 특징으로 올바른 것은?**

A) 영구적인 액세스 키와 시크릿 키를 가진다  
B) 임시 보안 자격 증명을 제공하며, AWS 서비스나 애플리케이션이 주로 사용한다  
C) 사용자와 동일하게 콘솔 로그인이 가능하다  
D) 다른 역할을 포함할 수 있다 (중첩 가능)  

<details>
<summary>정답 및 해설</summary>

**정답: B**

**해설:**
- **B (정답)**: IAM 역할의 핵심 특징입니다. 역할은 영구적인 자격 증명이 없으며, 필요할 때 임시 보안 자격 증명을 제공합니다. 주로 EC2 인스턴스, Lambda 함수 등 AWS 서비스나 애플리케이션이 사용합니다.

- **A (오답)**: 역할은 영구적인 자격 증명을 가지지 않습니다. 이는 사용자(User)의 특징입니다.

- **C (오답)**: 역할은 직접 콘솔 로그인이 불가능합니다. 사용자가 역할을 "assume"하여 사용합니다.

- **D (오답)**: 역할은 다른 역할을 포함할 수 없습니다. 이는 그룹도 마찬가지입니다.

**관련 개념**: IAM 역할 특성, 임시 자격 증명, AssumeRole
</details>

## 문제 5: IAM 보안 모범 사례 (종합 이해)

**회사에서 AWS 환경의 보안을 강화하려고 합니다. 다음 중 IAM 보안 모범 사례에 해당하지 않는 것은?**

A) 루트 계정에 MFA(Multi-Factor Authentication)를 설정한다  
B) 모든 개발자에게 동일한 IAM 사용자 계정을 공유하여 사용한다  
C) 최소 권한 원칙을 적용하여 필요한 최소한의 권한만 부여한다  
D) 사용하지 않는 액세스 키는 정기적으로 삭제하거나 비활성화한다  

<details>
<summary>정답 및 해설</summary>

**정답: B**

**해설:**
- **B (정답/잘못된 사례)**: 여러 사용자가 하나의 IAM 계정을 공유하는 것은 보안상 매우 위험합니다. 개별 책임 추적이 불가능하고, 한 사람이 퇴사할 때 모든 사람의 액세스에 영향을 줍니다. 각 사용자는 개별 IAM 계정을 가져야 합니다.

- **A (올바른 사례)**: 루트 계정 MFA 설정은 필수 보안 조치입니다.

- **C (올바른 사례)**: 최소 권한 원칙은 IAM의 핵심 보안 원칙입니다.

- **D (올바른 사례)**: 사용하지 않는 자격 증명 정리는 보안 위험을 줄입니다.

**추가 모범 사례**:
- 정기적인 권한 검토
- 그룹을 통한 권한 관리
- 강력한 비밀번호 정책
- CloudTrail을 통한 활동 모니터링

**관련 개념**: IAM 보안 모범 사례, 개별 계정 원칙, 책임 추적
</details>
