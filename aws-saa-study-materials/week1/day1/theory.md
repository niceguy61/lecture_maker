## 1. AWS란 무엇인가?

Amazon Web Services(AWS)는 아마존이 제공하는 클라우드 컴퓨팅 플랫폼입니다. 2006년에 시작되어 현재 전 세계에서 가장 널리 사용되는 클라우드 서비스 제공업체가 되었습니다.

### 클라우드 컴퓨팅의 핵심 개념

**클라우드 컴퓨팅**이란 인터넷을 통해 컴퓨팅 리소스(서버, 스토리지, 데이터베이스, 네트워킹, 소프트웨어 등)를 온디맨드로 제공하는 서비스입니다.

#### 전통적인 IT 인프라 vs 클라우드

**전통적인 방식의 문제점:**
- 높은 초기 투자 비용 (서버, 네트워크 장비 구매)
- 용량 계획의 어려움 (과소/과대 프로비저닝)
- 유지보수 및 관리 부담
- 확장성의 한계

**클라우드의 장점:**
- **비용 효율성**: 사용한 만큼만 지불 (Pay-as-you-go)
- **탄력성**: 필요에 따라 즉시 확장/축소
- **글로벌 접근성**: 전 세계 어디서나 서비스 배포
- **안정성**: 높은 가용성과 내결함성
- **보안**: 엔터프라이즈급 보안 기능

### AWS의 6가지 장점

1. **자본 비용을 가변 비용으로 전환**
   - 서버 구매 대신 사용량에 따른 요금 지불

2. **규모의 경제로 인한 이익**
   - AWS의 대규모 운영으로 더 저렴한 비용

3. **용량 추정 불필요**
   - 필요에 따라 자동으로 확장/축소

4. **속도와 민첩성 향상**
   - 몇 분 만에 전 세계에 리소스 배포

5. **데이터 센터 운영 및 유지 관리 비용 절약**
   - 인프라 관리에서 해방되어 비즈니스에 집중

6. **몇 분 만에 전 세계에 배포**
   - 글로벌 인프라를 통한 빠른 서비스 확장

## 2. AWS 글로벌 인프라

AWS는 전 세계에 분산된 물리적 인프라를 통해 서비스를 제공합니다. 이 인프라는 계층적 구조로 설계되어 있습니다.

### 2.1 리전 (Regions)

**리전**은 AWS가 데이터 센터를 클러스터링하는 물리적 위치입니다.

#### 리전의 특징
- 전 세계 33개 리전 운영 (2024년 기준)
- 각 리전은 지리적으로 분리되어 있음
- 리전 간에는 완전히 독립적
- 대부분의 AWS 서비스는 리전별로 운영

#### 리전 선택 기준

1. **규정 준수 (Compliance)**
   - 데이터 거버넌스 및 법적 요구사항
   - 예: 한국 데이터는 한국 내에서만 처리

2. **지연 시간 (Latency)**
   - 사용자와 가까운 리전 선택
   - 네트워크 지연 시간 최소화

3. **서비스 가용성 (Service Availability)**
   - 모든 리전에서 모든 서비스를 제공하지 않음
   - 필요한 서비스가 제공되는 리전 확인

4. **비용 (Pricing)**
   - 리전별로 가격이 다름
   - 일반적으로 미국 동부가 가장 저렴

### 2.2 가용 영역 (Availability Zones, AZ)

**가용 영역**은 리전 내의 하나 이상의 개별 데이터 센터입니다.

#### 가용 영역의 특징
- 각 리전은 최소 3개, 최대 6개의 AZ 보유
- AZ 간 거리: 100km 이내 (지연 시간 최소화)
- 각 AZ는 독립적인 전력, 냉각, 물리적 보안
- 고속 네트워크로 연결 (지연 시간 < 1ms)

#### 가용 영역 명명 규칙
```
리전코드 + 알파벳
예: us-east-1a, us-east-1b, ap-northeast-2a
```

#### 고가용성 설계
- **단일 AZ**: 단일 장애점 존재
- **다중 AZ**: 한 AZ 장애 시에도 서비스 지속
- **권장사항**: 최소 2개 AZ에 리소스 분산 배치

### 2.3 엣지 로케이션 (Edge Locations)

**엣지 로케이션**은 콘텐츠를 캐싱하여 사용자에게 더 빠른 배송을 제공하는 지점입니다.

#### 엣지 로케이션의 특징
- 전 세계 400개 이상의 엣지 로케이션
- CloudFront CDN 서비스의 핵심 구성 요소
- 리전보다 사용자에게 더 가까이 위치
- 정적 콘텐츠 캐싱 및 동적 콘텐츠 가속화

#### 주요 서비스
- **CloudFront**: 글로벌 CDN 서비스
- **Route 53**: DNS 서비스
- **AWS Global Accelerator**: 네트워크 성능 최적화

### 2.4 리전별 엣지 캐시 (Regional Edge Caches)

- 엣지 로케이션과 오리진 서버 사이의 중간 캐시
- 더 큰 캐시 용량으로 캐시 적중률 향상
- 자주 액세스되지 않는 콘텐츠 저장

## 3. AWS 서비스 카테고리

AWS는 200개 이상의 서비스를 제공하며, 주요 카테고리는 다음과 같습니다.

### 3.1 컴퓨팅 서비스
- **EC2**: 가상 서버 인스턴스
- **Lambda**: 서버리스 컴퓨팅
- **ECS/EKS**: 컨테이너 서비스
- **Batch**: 배치 컴퓨팅

### 3.2 스토리지 서비스
- **S3**: 객체 스토리지
- **EBS**: 블록 스토리지
- **EFS**: 파일 스토리지
- **Glacier**: 아카이브 스토리지

### 3.3 데이터베이스 서비스
- **RDS**: 관계형 데이터베이스
- **DynamoDB**: NoSQL 데이터베이스
- **ElastiCache**: 인메모리 캐시
- **Redshift**: 데이터 웨어하우스

### 3.4 네트워킹 서비스
- **VPC**: 가상 프라이빗 클라우드
- **CloudFront**: CDN 서비스
- **Route 53**: DNS 서비스
- **Direct Connect**: 전용 네트워크 연결

### 3.5 보안 및 자격 증명
- **IAM**: 자격 증명 및 액세스 관리
- **KMS**: 키 관리 서비스
- **CloudTrail**: API 호출 로깅
- **GuardDuty**: 위협 탐지

### 3.6 관리 및 모니터링
- **CloudWatch**: 모니터링 서비스
- **CloudFormation**: 인프라 as 코드
- **Systems Manager**: 시스템 관리
- **Config**: 리소스 구성 관리

## 4. AWS 공동 책임 모델

AWS와 고객 간의 보안 책임을 명확히 구분하는 모델입니다.

### AWS의 책임 ("클라우드의 보안")
- 물리적 인프라 보안
- 하드웨어, 소프트웨어, 네트워킹, 시설
- 관리형 서비스의 구성 및 취약점 패치

### 고객의 책임 ("클라우드에서의 보안")
- 게스트 운영 체제 및 업데이트
- 애플리케이션 소프트웨어 및 유틸리티
- 보안 그룹 구성
- 방화벽 구성
- 네트워크 및 플랫폼 수준 자격 증명 관리
- 데이터 암호화

### 서비스 유형별 책임 분담

#### IaaS (Infrastructure as a Service)
- 예: EC2, VPC
- 고객이 더 많은 보안 책임

#### PaaS (Platform as a Service)
- 예: RDS, Lambda
- AWS와 고객이 책임 공유

#### SaaS (Software as a Service)
- 예: WorkMail, WorkDocs
- AWS가 대부분의 보안 책임

## 5. AWS 계정 및 결제

### 5.1 AWS 계정 구조
- **루트 계정**: 모든 권한을 가진 최상위 계정
- **IAM 사용자**: 제한된 권한을 가진 개별 사용자
- **조직 (Organizations)**: 여러 계정을 중앙에서 관리

### 5.2 AWS 프리 티어
- **12개월 무료**: EC2, S3, RDS 등 주요 서비스
- **항상 무료**: Lambda, DynamoDB 등 일부 서비스
- **평가판**: 단기간 무료 사용

### 5.3 결제 모델
- **온디맨드**: 사용한 만큼 지불
- **예약 인스턴스**: 1-3년 약정으로 할인
- **스팟 인스턴스**: 경매 방식으로 저렴한 가격

## 6. AWS Well-Architected Framework 소개

AWS에서 제안하는 5가지 설계 원칙:

### 6.1 운영 우수성 (Operational Excellence)
- 시스템 운영 및 모니터링
- 지속적인 개선

### 6.2 보안 (Security)
- 정보 및 시스템 보호
- 위험 평가 및 완화 전략

### 6.3 안정성 (Reliability)
- 장애 복구 능력
- 동적 수요 충족

### 6.4 성능 효율성 (Performance Efficiency)
- 컴퓨팅 리소스의 효율적 사용
- 기술 발전에 따른 효율성 유지

### 6.5 비용 최적화 (Cost Optimization)
- 불필요한 비용 방지
- 최저 가격으로 비즈니스 목표 달성
