## 문제 1: AWS 글로벌 인프라 구조

**문제**: AWS 글로벌 인프라의 계층 구조를 올바르게 나타낸 것은?

A) 리전 → 엣지 로케이션 → 가용 영역  
B) 가용 영역 → 리전 → 엣지 로케이션  
C) 리전 → 가용 영역 → 데이터 센터  
D) 엣지 로케이션 → 리전 → 가용 영역  

<details>
<summary>정답 및 해설</summary>

**정답: C) 리전 → 가용 영역 → 데이터 센터**

**해설**:
AWS 글로벌 인프라는 다음과 같은 계층 구조를 가집니다:
- **리전 (Region)**: 지리적으로 분리된 AWS 데이터 센터 클러스터 영역
- **가용 영역 (Availability Zone)**: 리전 내의 하나 이상의 개별 데이터 센터
- **데이터 센터**: 실제 물리적 서버와 네트워크 장비가 위치한 시설

엣지 로케이션은 별도의 CDN 인프라로, 이 계층 구조와는 독립적으로 운영됩니다.

**관련 개념**: 
- 각 리전은 최소 3개의 가용 영역을 보유
- 가용 영역 간 거리는 100km 이내로 지연 시간 최소화
- 엣지 로케이션은 CloudFront CDN 서비스를 위한 별도 인프라
</details>

## 문제 2: 리전 선택 기준

**문제**: 새로운 웹 애플리케이션을 배포할 AWS 리전을 선택할 때 고려해야 할 가장 중요한 요소들은? (복수 선택)

A) 규정 준수 및 데이터 거버넌스 요구사항  
B) 사용자와의 지리적 거리 (지연 시간)  
C) 필요한 AWS 서비스의 가용성  
D) 리전별 가격 차이  
E) 리전의 설립 연도  

<details>
<summary>정답 및 해설</summary>

**정답: A, B, C, D (E 제외)**

**해설**:
리전 선택 시 고려해야 할 4가지 주요 기준:

1. **규정 준수 (Compliance)**: 
   - 데이터 거버넌스 및 법적 요구사항
   - 예: 개인정보보호법에 따른 국내 데이터 처리 의무

2. **지연 시간 (Latency)**:
   - 사용자와 가까운 리전 선택으로 네트워크 지연 최소화
   - 사용자 경험 향상

3. **서비스 가용성 (Service Availability)**:
   - 모든 리전에서 모든 서비스를 제공하지 않음
   - 필요한 서비스가 제공되는 리전 확인 필요

4. **비용 (Pricing)**:
   - 리전별로 가격이 다름
   - 일반적으로 미국 동부(us-east-1)가 가장 저렴

**E는 틀린 이유**: 리전의 설립 연도는 서비스 품질이나 성능과 직접적인 관련이 없습니다.
</details>

## 문제 3: 가용 영역의 특징

**문제**: AWS 가용 영역(Availability Zone)에 대한 설명 중 올바른 것은?

A) 각 리전은 반드시 2개의 가용 영역만 보유한다  
B) 가용 영역 간 거리는 1000km 이내로 설계된다  
C) 가용 영역은 독립적인 전력, 냉각, 물리적 보안을 가진다  
D) 모든 가용 영역은 동일한 데이터 센터를 공유한다  

<details>
<summary>정답 및 해설</summary>

**정답: C) 가용 영역은 독립적인 전력, 냉각, 물리적 보안을 가진다**

**해설**:
가용 영역의 올바른 특징:
- **독립성**: 각 AZ는 독립적인 전력, 냉각, 물리적 보안 시설을 보유
- **내결함성**: 한 AZ의 장애가 다른 AZ에 영향을 주지 않도록 설계
- **고속 연결**: AZ 간 고속 네트워크로 연결 (지연 시간 < 1ms)

**틀린 선택지 해설**:
- A) 각 리전은 **최소 3개, 최대 6개**의 AZ를 보유
- B) AZ 간 거리는 **100km 이내** (1000km가 아님)
- D) 각 AZ는 **하나 이상의 개별 데이터 센터**로 구성 (공유하지 않음)

**실무 적용**: 고가용성을 위해 최소 2개 AZ에 리소스를 분산 배치하는 것이 권장됩니다.
</details>

## 문제 4: AWS 공동 책임 모델

**문제**: AWS 공동 책임 모델에서 고객의 책임에 해당하는 것은?

A) 물리적 데이터 센터 보안  
B) 하드웨어 인프라 유지보수  
C) 게스트 운영 체제 패치 및 업데이트  
D) AWS 서비스의 소프트웨어 패치  

<details>
<summary>정답 및 해설</summary>

**정답: C) 게스트 운영 체제 패치 및 업데이트**

**해설**:
AWS 공동 책임 모델의 구분:

**AWS 책임 ("클라우드의 보안")**:
- 물리적 인프라 보안 (A)
- 하드웨어, 소프트웨어, 네트워킹, 시설 (B)
- 관리형 서비스의 구성 및 취약점 패치 (D)

**고객 책임 ("클라우드에서의 보안")**:
- 게스트 운영 체제 및 업데이트 ✓
- 애플리케이션 소프트웨어 및 유틸리티
- 보안 그룹 구성
- 방화벽 구성
- 네트워크 및 플랫폼 수준 자격 증명 관리
- 데이터 암호화

**서비스 유형별 차이**:
- **IaaS** (EC2): 고객이 더 많은 책임
- **PaaS** (RDS): AWS와 고객이 책임 공유
- **SaaS** (WorkMail): AWS가 대부분의 책임
</details>

## 문제 5: AWS 서비스 카테고리

**문제**: 다음 AWS 서비스와 해당 카테고리의 매칭이 올바른 것은?

A) Lambda - 컴퓨팅, S3 - 스토리지, RDS - 데이터베이스, VPC - 네트워킹  
B) EC2 - 스토리지, DynamoDB - 컴퓨팅, CloudFront - 데이터베이스, IAM - 네트워킹  
C) EBS - 컴퓨팅, Lambda - 스토리지, Route 53 - 데이터베이스, S3 - 네트워킹  
D) CloudWatch - 컴퓨팅, Glacier - 네트워킹, ElastiCache - 스토리지, ECS - 데이터베이스  

<details>
<summary>정답 및 해설</summary>

**정답: A) Lambda - 컴퓨팅, S3 - 스토리지, RDS - 데이터베이스, VPC - 네트워킹**

**해설**:
AWS 서비스의 올바른 카테고리 분류:

**컴퓨팅 서비스**:
- **Lambda**: 서버리스 컴퓨팅 ✓
- EC2: 가상 서버 인스턴스
- ECS/EKS: 컨테이너 서비스

**스토리지 서비스**:
- **S3**: 객체 스토리지 ✓
- EBS: 블록 스토리지
- EFS: 파일 스토리지
- Glacier: 아카이브 스토리지

**데이터베이스 서비스**:
- **RDS**: 관계형 데이터베이스 ✓
- DynamoDB: NoSQL 데이터베이스
- ElastiCache: 인메모리 캐시

**네트워킹 서비스**:
- **VPC**: 가상 프라이빗 클라우드 ✓
- CloudFront: CDN 서비스
- Route 53: DNS 서비스

**틀린 선택지들은 서비스와 카테고리가 잘못 매칭되어 있습니다.**
</details>
