## 문제 1: 클라우드 서비스 모델 (4점)

**다음 중 AWS Lambda가 해당하는 클라우드 서비스 모델은 무엇입니까?**

A) IaaS (Infrastructure as a Service)  
B) PaaS (Platform as a Service)  
C) SaaS (Software as a Service)  
D) DaaS (Desktop as a Service)

<details>
<summary>정답 및 해설 보기</summary>

**정답: B) PaaS (Platform as a Service)**

**해설:**
- **Lambda는 PaaS 모델**에 해당합니다. 서버 관리 없이 코드만 업로드하면 실행 환경을 제공하는 서버리스 컴퓨팅 서비스입니다.
- **IaaS**: EC2, S3 등 인프라 리소스를 제공 (사용자가 OS, 미들웨어 관리)
- **PaaS**: Lambda, RDS, Elastic Beanstalk 등 플랫폼을 제공 (사용자는 애플리케이션만 관리)
- **SaaS**: WorkMail, Chime 등 완성된 소프트웨어를 제공

**관련 개념:**
- 서버리스 컴퓨팅
- 공동 책임 모델에서 AWS와 고객의 책임 분담
</details>

## 문제 2: AWS 글로벌 인프라 (4점)

**AWS 리전(Region)에 대한 설명으로 올바른 것은?**

A) 각 리전은 최소 2개의 가용 영역으로 구성된다  
B) 리전 간 데이터는 자동으로 복제된다  
C) 모든 AWS 서비스는 모든 리전에서 동일하게 제공된다  
D) 각 리전은 최소 3개의 가용 영역으로 구성된다

<details>
<summary>정답 및 해설 보기</summary>

**정답: D) 각 리전은 최소 3개의 가용 영역으로 구성된다**

**해설:**
- **AWS 리전은 최소 3개의 가용 영역(AZ)**으로 구성되어 고가용성을 보장합니다.
- **A) 틀림**: 최소 3개의 AZ로 구성 (2개가 아님)
- **B) 틀림**: 데이터는 명시적 허가 없이 리전 간 이동하지 않음 (데이터 주권 보장)
- **C) 틀림**: 신규 서비스는 주로 us-east-1에서 먼저 출시되며, 모든 서비스가 모든 리전에서 제공되지는 않음

**관련 개념:**
- 가용 영역(Availability Zone)
- 고가용성 아키텍처 설계
- 데이터 주권 및 규정 준수
</details>

## 문제 3: 가용 영역 (4점)

**다음 중 AWS 가용 영역(Availability Zone)에 대한 설명으로 틀린 것은?**

A) 각 가용 영역은 독립적인 전력, 냉각, 네트워킹 시설을 가진다  
B) 가용 영역 간 네트워크 지연시간은 10ms 미만이다  
C) 가용 영역은 서로 다른 리전에 위치한다  
D) 단일 장애점을 제거하기 위해 물리적으로 분리되어 있다

<details>
<summary>정답 및 해설 보기</summary>

**정답: C) 가용 영역은 서로 다른 리전에 위치한다**

**해설:**
- **가용 영역은 같은 리전 내에서** 물리적으로 분리된 데이터 센터입니다.
- **A) 맞음**: 각 AZ는 독립적인 인프라를 가져 단일 장애점을 제거
- **B) 맞음**: AZ 간 고속 네트워크로 연결되어 지연시간이 10ms 미만
- **D) 맞음**: 물리적 분리를 통해 재해나 장애 시 영향을 최소화

**관련 개념:**
- Multi-AZ 배포 전략
- 고가용성 및 내결함성 설계
- 리전 vs 가용 영역의 차이점
</details>

## 문제 4: 리전 선택 기준 (4점)

**한국의 스타트업이 한국 사용자를 대상으로 하는 웹 서비스를 AWS에서 운영하려고 합니다. 리전 선택 시 가장 우선적으로 고려해야 할 요소는?**

A) 비용 (Cost)  
B) 지연시간 (Latency)  
C) 서비스 가용성 (Service Availability)  
D) 법적 요구사항 (Compliance)

<details>
<summary>정답 및 해설 보기</summary>

**정답: B) 지연시간 (Latency)**

**해설:**
- **한국 사용자 대상 서비스**의 경우 **지연시간이 가장 중요한 요소**입니다.
- 서울 리전(ap-northeast-2) 사용 시 한국 사용자에게 최적의 성능을 제공할 수 있습니다.
- **다른 요소들의 중요도:**
  - **비용**: 중요하지만 사용자 경험보다 우선순위가 낮음
  - **서비스 가용성**: 대부분의 주요 서비스는 서울 리전에서 제공
  - **법적 요구사항**: 특별한 규제가 없는 일반적인 웹 서비스의 경우 우선순위가 낮음

**관련 개념:**
- 리전별 지연시간 비교
- 사용자 경험 최적화
- 글로벌 서비스 배포 전략
</details>

## 문제 5: AWS 공동 책임 모델 (4점)

**AWS 공동 책임 모델에서 고객의 책임 영역에 해당하는 것은?**

A) 데이터 센터의 물리적 보안  
B) 하이퍼바이저 패치 및 업데이트  
C) 게스트 운영체제의 보안 패치  
D) AWS 글로벌 네트워크 인프라

<details>
<summary>정답 및 해설 보기</summary>

**정답: C) 게스트 운영체제의 보안 패치**

**해설:**
- **공동 책임 모델**에서 고객은 "클라우드 내부(Security IN the Cloud)"의 보안을 담당합니다.
- **고객 책임 영역:**
  - 게스트 OS 패치 및 업데이트
  - 애플리케이션 보안
  - 데이터 암호화
  - 네트워크 방화벽 설정
  - IAM 사용자 및 권한 관리

- **AWS 책임 영역 (틀린 선택지들):**
  - **A) 물리적 보안**: AWS가 데이터 센터 보안 담당
  - **B) 하이퍼바이저**: AWS가 가상화 인프라 관리
  - **D) 글로벌 네트워크**: AWS가 네트워크 인프라 관리

**관련 개념:**
- Security OF the Cloud vs Security IN the Cloud
- 서비스 모델별 책임 분담 (IaaS, PaaS, SaaS)
</details>
